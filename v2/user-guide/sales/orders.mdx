Orders are the heart of your e-commerce business. This guide covers how to manage orders from placement to fulfillment.

## Overview

The Orders section allows you to:

- View all incoming orders
- Track order status
- Process payments and refunds
- Manage shipping and fulfillment
- Communicate with customers

## Order List View

The order list shows key information at a glance:

| Column | Description |
|--------|-------------|
| **Order number** | Unique identifier for the order |
| **Date** | When the order was placed |
| **Customer** | Customer name and email |
| **Total** | Order total amount |
| **Payment status** | Paid, pending, refunded, etc. |
| **Fulfillment status** | Unfulfilled, fulfilled, partially fulfilled |

### Order Information

The order detail page shows:

- **Order items** - Products, quantities, and prices
- **Customer information** - Name, email, phone
- **Shipping address** - Delivery destination
- **Billing address** - Payment address
- **Payment details** - Method and status
- **Order timeline** - History of order events

## Order Status

Orders progress through different statuses:

### Payment Status

| Status | Description |
|--------|-------------|
| **Pending** | Awaiting payment |
| **Paid** | Payment received |
| **Partially refunded** | Some items refunded |
| **Refunded** | Full refund issued |

### Fulfillment Status

| Status | Description |
|--------|-------------|
| **Unfulfilled** | Order not yet shipped |
| **Partially fulfilled** | Some items shipped |
| **Fulfilled** | All items shipped |

## Processing an Order

### Step 1: Review the Order

1. Open the order details
2. Verify the items and quantities
3. Check the shipping address

### Step 2: Prepare for Shipping

1. Pick and pack the items
2. Generate shipping labels if integrated
3. Record tracking information

### Step 3: Mark as Fulfilled

1. Click **Fulfill order**
2. Add tracking number (optional)
3. Notify customer via email

## Order Filtering

Filter orders to find what you need:

- **Date range** - Orders from a specific period
- **Status** - Payment or fulfillment status
- **Customer** - Orders from a specific customer
- **Amount** - Orders above or below a value
